The Logistic Regression is a statistical model that makes use of Input data as a
Training-set. This means the algorithm learns the connection between one or more
independent variables (multi-class) and their assigned outcome value. The outcome value
must belong to one of two classes, therefore it gets a categorical value of 0 or 1. The
goal of the Logistic Regression is then to recognize based on the correlation on the
Training-set to find the best fit for a new Test-set. \\
\\
\section{Mathematical background of the Logistic Regression}
Like a linear regression, the aim is to fit a curve through the data in a way that all
points are divided into two categories. Since the Logistic Regression treats the Input
data for 166 models one by one, all mathematical approaches are simplified to the case
$\mathbb{R}^1$ definition. The so called \textbf{sigmoid function} is then represented by
the following equation:
\begin{align}
\sigma: \mathbb{R} \to [0,1], \,\,\,\, \sigma(t) := \frac{1}{1 + e^{-t}}
\label{Sigmoid function}
\end{align}
Furthermore it has a turning point at h(x) = 0.5 for x = 0. The Logistic Regression puts
every point above the turning point in the first category mostly represented by 1 and
every point below in the second category assigned by 0. This is done through the following
consideration:\\
If we applied a linear regression on the data set it would minimize the residuals between
the data points and a straight line. The mathematical definition is given as follows:
\begin{definition} Linear Regression \\
Considering some data \{x\textsubscript{1}, ..., x\textsubscript{n}\}, a linear
    regression y: $\mathbb{R} \to \mathbb{R}$ is given by
\begin{align}
y = y(x) = \beta\textsubscript{0} + \sum_{i = 1}^{n} \beta\textsubscript{i}
        x\textsubscript{i} = \beta\textsubscript{0} + \beta\textsubscript{1}
        x\textsubscript{1} + \dots + \beta\textsubscript{n} x\textsubscript{n} =
        \beta^\intercal x
\label{Linear Regression Formula}
\end{align}
Thereby $\beta\textsubscript{i} \,\,\, \forall i$ are regression coefficients to
    determine the best fit through the data.
\label{Linear regression}
\end{definition}
Although the linear regression seems to do exactly what is needed it can not be chosen.
The problem is that the equation (\ref{Linear Regression Formula}) is divergent and does
not return two categories. Therefore one wants to fit a convergent curve instead of a
straight line (\ref{Sigmoid function}) through all data points. Thus the approach of
combining the Linear regression (\ref{Linear Regression Formula}) with the sigmoid
function (\ref{Sigmoid function}) is a useful idea. This leads to:
\begin{align}
h(x) := \sigma(y(x)) = \sigma(\beta^\intercal x) = \frac{1}{1 + e^{-\beta^\intercal
    x}} = \frac{1}{1 + e^{-(\beta\textsubscript{0} + \sum_{i = 1}^{n}
    \beta\textsubscript{i} x\textsubscript{i})}}
\label{Sigmoid + Linear Regression}
\end{align}

5.1 Approaching the problem
Up to this point the main goal of this study has been to numerically solve all ODEs of
166 models using a collection of fourty different setting combinations and investigate the
resulting simulation times. Thus for all of these models we can say precisely what solver
combination can simulate and compute all ODDEs in the shortest amount of time.
But what about other models? The problem is that none of the collected data can
be applied to a new model that is slightly different from all already examined models.
The first idea could be to do the whole study again. This however would be highly
impracticable if one deals with more than a couple of models. Also guessing the perfect
setting by hand based on the existing data is not an option because the ODEs can be
unpredictable. Since the behaviour of solving those equations does not only depend on
the size of the model but also on intrinsic yet unclassified components, we need the data
and the already analysed models.
A common approach to "predict" the perfect simulation setting is to utilize a
classifi-
cation algorithm
. Such an algorithm uses input data to classify new data. Different
types of classification algorithms are for example Naive Bayes Classifier, Decision Trees,
Neural Networks or the
Logistic Regression
.
The Logistic Regression is a statistical model that makes use of Input data as a Training-
set. This means the algorithm learns the connection between one or more independent
variables (multi-class) and their assigned outcome value. The outcome value must be-
long to one of two classes, therefore it gets a categorical value of 0 or 1. The goal of the
Logistic Regression is then to recognize based on the correlation on the Training-set to
find the best fit for a new Test-set.

Using the complete data, the Logistic Regression can

solution one would have to do the simulation study (chapter 4) for all 40 settings.
Yet it is also possible to "re-use" models of the former study because here all results
do already exist. The first idea could be to split the
training set
containing the input
matrix and the output vector in two parts, e.g. at a ratio of 4 : 1. The larger sample
becomes the new training set and the small matrix is called the
testing set

the testing set is then predicted and compared to the output of the testing set. Since
it is random which models belong to the training - and testing set and all models are
independent this approach is unbiased.
The only problem is that the prediction of the test set will most likely fit the result of the
training set better than those of the test set. This statement can be verified using the
following example: Let the testing set consist mostly of large models and the training
set almost entirely of small models. Then the Logistic Regression will not be able to
predict the correct result because there is no correlation between large models and small
models in the training set. For that reason it will take a result closer to the training set
than the testing set with a higher probability.

6 Outlook and possibilities
The whole bachelor thesis was meant to investigate the influence of different settings
on solving ODEs numerically. These settings were absolute and relative tolerances, the
linear solver and the multistep method. The main focus was to get results on the success
rate and the simulation time for a combination of those settings. Afterwards this data
was used to create a model which was able to predict the best setting for different goals.
Although this study is a good basis and contains useful answers it can be extended.
During the study certain aspects were restricted while others were left out completely
due to the time constraint of a bachelor thesis. For that reason this chapter is aligned
to shortly discuss further possibilities for the future.
One of the first choices was to use the solver CVODE of the SUNDIALS toolbox. As men-
tioned in chapter 3 two other possibilities were available,
CVODES
and
ARKODE
.
Compared to CVODE, CVODES has one additional feature. It can solve ODEs by ap-
plying two different types, forward and adjoint sensitivity. Thereby an analysis of ODEs
through sensitivity "quanti£es the relationship between changes in model parameters
and changes in model outputs" ([2], p.1). This can be used to for estimating unknown
parameters in an ODE that can not be measured experimentally. This problem was
already mentioned in chapter 1 and could extend the study by another topic.
ARKODE on the other hand uses a completely different approach to solve the IVP. It
applies a combination of two single step
Runge Kutta
methods, an explicit and an
implicit one. Since both BDF and Adams-Moulton are implicit multistep methods it
would be interesting to compare their impact on solving an ODE with the remaining
settings.
During the study the kinetic law of the ODEs were investigated. This was done to
add a parameter that can influence the simulation time of the ODE. Another possibil-
ity would be to investigate
topology loops
. These are loops between species which
interact through the ODEs. The conjecture would be that topology loops increase the
stiffness
of an ODE. This could lead to a change of optimal settings for some models.
It is known that the BDF method can solve stiff differential equations more successfully
than Adams-Moulton because BDF has a better stability.
The stiffness of an ODE itself could an interesting new parameter to investigate. The
main problem is that there does not exist a unique classification when an ODE is stiff.
Therefore a somewhat arbitrary choice in definition would have to be made.
